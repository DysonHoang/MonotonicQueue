01 — Concept

Khái niệm cơ bản

Monotonic Queue (Hàng đợi đơn điệu) là một cấu trúc dữ liệu dạng deque (double-ended queue) được duy trì sao cho các phần tử trong đó luôn theo thứ tự tăng dần hoặc giảm dần.

💡 Định nghĩa
Monotonic Queue là deque mà khi thêm phần tử mới vào, ta loại bỏ các phần tử đầu/cuối vi phạm tính đơn điệu trước khi chèn — đảm bảo deque luôn có thứ tự nhất quán.

Hai biến thể

Tăng dần (Monotonic Increasing) — phần tử nhỏ nhất nằm ở đầu deque. Dùng để tìm minimum trong cửa sổ.

Giảm dần (Monotonic Decreasing) — phần tử lớn nhất nằm ở đầu deque. Dùng để tìm maximum trong cửa sổ.

⚠️ Lưu ý
Ta thường lưu chỉ số (index) thay vì giá trị vào deque, để kiểm tra xem phần tử có còn trong cửa sổ hay không.

03 — Implementation

Cài đặt & Code

Python

pythonfrom collections import deque

def sliding_window_maximum(nums: list, k: int) -> list:
    dq = deque()   # lưu index, deque giảm dần theo giá trị
    result = []

    for i in range(len(nums)):
        # 1. Loại phần tử ngoài cửa sổ (index quá cũ)
        while dq and dq[0] <= i - k:
            dq.popleft()

        # 2. Loại phần tử nhỏ hơn phần tử hiện tại (từ đuôi)
        while dq and nums[dq[-1]] <= nums[i]:
            dq.pop()

        # 3. Thêm index hiện tại vào đuôi
        dq.append(i)

        # 4. Đầu deque là max của cửa sổ
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

C++

c++#include <deque>
#include <vector>
using namespace std;

vector<int> slidingWindowMax(vector<int>& nums, int k) {
    deque<int> dq;   // lưu index
    vector<int> result;

    for (int i = 0; i < nums.size(); i++) {
        // Loại phần tử ngoài cửa sổ
        while (!dq.empty() && dq.front() <= i - k)
            dq.pop_front();

        // Duy trì tính đơn điệu giảm dần
        while (!dq.empty() && nums[dq.back()] <= nums[i])
            dq.pop_back();

        dq.push_back(i);

        if (i >= k - 1)
            result.push_back(nums[dq.front()]);
    }
    return result;
}
✅ Mẫu tổng quát
Mọi bài toán Monotonic Queue đều lặp lại 4 bước: (1) pop front nếu out-of-window, (2) pop back nếu vi phạm đơn điệu, (3) push back index mới, (4) ghi nhận kết quả từ front.
04 — Classic Problem

Sliding Window Maximum (LeetCode 239)

Đây là bài toán kinh điển nhất cho Monotonic Queue:

Đề: Cho mảng nums và số nguyên k. Trả về mảng chứa giá trị lớn nhất của mỗi cửa sổ trượt có độ rộng k.

Ví dụ minh hoạ
Cửa sổ Các phần tử Maximum
[0..2] 3, 1, 4 4
[1..3] 1, 4, 1 4
[2..4] 4, 1, 5 5
[3..5] 1, 5, 9 9
[4..6] 9, 2, 6 (vẫn 9) 9
[5..7] 9, 2, 6 9

Kết quả: [4, 4, 5, 9, 9, 9]

Tại sao không dùng brute force?

Brute force O(n·k) sẽ TLE với n = 10⁵, k = 5×10⁴. Monotonic Queue cho O(n) vì mỗi phần tử chỉ được push/pop đúng một lần.

🎯 Pattern nhận biết
Khi bài toán có dạng "tìm min/max trong đoạn [i-k, i]" hoặc DP với dạng dp[i] = max(dp[j]) + f(i) với j trong khoảng cố định → nghĩ ngay đến Monotonic Queue.
06 — Complexity

Độ phức tạp

Phương pháp Thời gian Không gian
Brute Force O(n·k) O(1)
Segment Tree / Sparse Table O(n log n) O(n log n)
Monotonic Queue O(n) O(k)

Lý do O(n): Mỗi phần tử được thêm vào deque đúng 1 lần và bị loại đúng tối đa 1 lần. Tổng số thao tác trên deque là O(2n) = O(n).

📌 Tóm tắt
Monotonic Queue là công cụ cực mạnh khi cần truy vấn min/max trong cửa sổ trượt với độ phức tạp tối ưu O(n). Nắm vững cấu trúc này sẽ mở ra một lớp bài toán DP tối ưu hoá quan trọng trong competitive programming.
